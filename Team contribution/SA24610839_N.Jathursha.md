## Function Overview
Responsible for managing the doctors registered in the hospital system and their weekly availability schedules. This module allows admins to onboard doctors, maintain their specialist profiles, and configure the time slots during which each doctor is available for appointments ‚Äî directly feeding the appointment booking system.

---

## CRUD Operations Covered

### CRUD 1 ‚Äî Doctor Profile Management
| Operation | Description | Files Involved |
|-----------|-------------|----------------|
| **Create** | Register a new doctor profile with specialization and ID card upload | `DoctorManagement.jsx`, `DoctorViewSet.create` in `views.py` |
| **Read** | View all doctors, filter by specialization, view individual doctor page | `DoctorManagement.jsx`, `DoctorViewSet.list` / `.retrieve` |
| **Update** | Edit doctor specialization, linked user account, and ID card | `DoctorManagement.jsx`, `DoctorViewSet.partial_update` |
| **Delete** | Remove a doctor profile from the system | `DoctorManagement.jsx`, `DoctorViewSet.destroy` |

### CRUD 2 ‚Äî Doctor Availability Scheduling
| Operation | Description | Files Involved |
|-----------|-------------|----------------|
| **Create** | Add availability slot (day of week, start time, end time) for a doctor | `Availability.jsx`, `DoctorAvailabilityViewSet.create` |
| **Read** | View all weekly slots for a doctor ‚Äî calendar/grid format | `Availability.jsx`, `DoctorAvailabilityViewSet.list` |
| **Update** | Modify existing slot times or toggle availability on/off | `Availability.jsx`, `DoctorAvailabilityViewSet.partial_update` |
| **Delete** | Remove an availability slot | `Availability.jsx`, `DoctorAvailabilityViewSet.destroy` |

## Database Connectivity
- **Model:** `Doctor` ‚Äî table `doctors` in `backend/api/models.py`  
  Fields: `user` (OneToOne FK ‚Üí User), `doctor_id_card`, `specialization`, `created_at`, `updated_at`
- **Model:** `DoctorAvailability` ‚Äî table `doctor_availability` in `backend/api/models.py`  
  Fields: `doctor` (FK ‚Üí Doctor), `day_of_week`, `start_time`, `end_time`, `is_available`  
  Unique constraint: `(doctor, day_of_week, start_time)`
- **Serializers:** `DoctorSerializer`, `DoctorAvailabilitySerializer` in `backend/api/serializers.py`
- **ViewSets:** `DoctorViewSet`, `DoctorAvailabilityViewSet` in `backend/api/views.py`
- **Endpoints:**  
  - `GET  /api/doctors/` ‚Äî List all doctors  
  - `POST /api/doctors/` ‚Äî Create doctor profile  
  - `GET  /api/doctors/{id}/` ‚Äî Get doctor details  
  - `PATCH /api/doctors/{id}/` ‚Äî Update doctor profile  
  - `DELETE /api/doctors/{id}/` ‚Äî Delete doctor  
  - `GET  /api/availability/` ‚Äî List availability slots  
  - `POST /api/availability/` ‚Äî Create availability slot  
  - `PATCH /api/availability/{id}/` ‚Äî Update slot  
  - `DELETE /api/availability/{id}/` ‚Äî Delete slot
