## Function Overview
Responsible for two interconnected modules: the AI-powered telephone call handling system (call intake, live call monitoring, call logs) and staff/receptionist record management. The call system uses Twilio for telephony and an AI/AGI receptionist layer (Whisper STT + Piper TTS + intent classifier) to handle incoming calls, book appointments, and dispatch emergencies automatically.

---

## CRUD Operations Covered

### CRUD 1 — Call Log Management
| Operation | Description | Files Involved |
|-----------|-------------|----------------|
| **Create** | Incoming call triggers creation of a call log entry with caller ID, intent, transcript | `twilio_views.py` (`twilio_incoming_call`), `call_ai_views.py` (`call_ai_intake`) |
| **Read** | View all call logs — filter by date, intent type, status; view transcript per call | `CallLogs.jsx`, `call_logs` view in `views.py` |
| **Update** | Update call status (Ongoing, Completed, Transferred, Missed) and notes | `CallConsole.jsx`, `twilio_status_callback` in `twilio_views.py` |
| **Delete** | Delete old/test call log entries from admin panel | `CallLogs.jsx` (admin delete action), `call_logs` view |

### CRUD 2 — Staff Record Management
| Operation | Description | Files Involved |
|-----------|-------------|----------------|
| **Create** | Register staff/receptionist profile linked to approved User account | `StaffManagement.jsx`, `Staff`/`Receptionist` models via `UserViewSet` |
| **Read** | View all staff members, their roles, status, and assigned shifts | `StaffManagement.jsx`, `staff_list` view in `views.py` |
| **Update** | Edit staff details, update role or operational status | `StaffManagement.jsx`, `UserViewSet.partial_update` |
| **Delete** | Deactivate or remove a staff record | `StaffManagement.jsx`, `UserViewSet.destroy` |

## Database Connectivity
- **Models:**  
  - `Staff` — table `staff` in `backend/api/models.py` (OneToOne → User)  
  - `Receptionist` — table `receptionists` in `backend/api/models.py` (OneToOne → User)  
  - Call session data stored via `call_session.py` and `mongodb_service.py`
- **Views:**  
  - `twilio_incoming_call`, `twilio_gather`, `twilio_status_callback`, `live_calls_snapshot`, `transfer_call` — in `backend/api/twilio_views.py`  
  - `call_ai_intake`, `call_ai_requests_list`, `call_ai_vocabulary` — in `backend/api/call_ai_views.py`  
  - `staff_list`, `call_logs` — in `backend/api/views.py`
- **WebSocket:** `consumers.py` — Django Channels consumer for real-time call status push to frontend
- **Endpoints:**  
  - `POST /api/twilio/voice/` — Twilio webhook — incoming call handler  
  - `POST /api/twilio/gather/` — Collect caller speech/DTMF input  
  - `POST /api/twilio/status/` — Call status callback  
  - `GET  /api/live-calls/` — Live calls snapshot  
  - `POST /api/calls/{call_sid}/transfer/` — Transfer call  
  - `POST /api/call-ai/intake/` — AI receptionist call intake  
  - `GET  /api/call-ai/requests/` — List AI-handled call requests  
  - `GET  /api/call-logs/` — All call logs  
  - `GET  /api/staff/` — Staff list

## Technologies Used
- **Frontend:** React 18, Tailwind CSS, React Query, WebSocket via `useSocket.js`
- **Backend:** Django REST Framework, Django Channels (WebSocket), Twilio REST API
- **AI Call Layer:** `call_ai/` module — Whisper STT (`stt_whisper.py`), Piper TTS (`tts_piper.py`), intent classifier (`intent_classifier.py`), slot extractor (`slot_extractor.py`), state machine (`state_machine.py`), AGI receptionist (`agi_receptionist.py`)
- **Telephony:** Twilio Programmable Voice (TwiML), Asterisk AGI integration
- **Database:** SQLite/PostgreSQL + optional MongoDB via `mongodb_service.py`
- **Real-time:** Django Channels + WebSocket (`consumers.py`, `routing.py`)
