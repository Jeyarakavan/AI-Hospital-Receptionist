## Function Overview
Responsible for the core appointment booking and patient management workflow. This includes recording patient details, scheduling appointments with specific doctors at specific date/time slots, tracking appointment statuses (Pending → Confirmed → Completed / Cancelled), and providing staff/admin with a full management interface.

---

## CRUD Operations Covered

### CRUD 1 — Appointment Booking & Management
| Operation | Description | Files Involved |
|-----------|-------------|----------------|
| **Create** | Book a new appointment — patient details, doctor, date, time, reason | `Appointments.jsx`, `AppointmentViewSet.create` in `views.py` |
| **Read** | View all appointments with filters (date, doctor, status), calendar view | `Appointments.jsx`, `AppointmentCard.jsx`, `AppointmentViewSet.list` |
| **Update** | Reschedule appointment, update status (Confirmed/Cancelled/Completed) | `Appointments.jsx`, `AppointmentViewSet.partial_update` |
| **Delete** | Cancel and remove an appointment record | `Appointments.jsx`, `AppointmentViewSet.destroy` |

### CRUD 2 — Patient Records Management
| Operation | Description | Files Involved |
|-----------|-------------|----------------|
| **Create** | Add a new patient record (name, age, phone, address, email) | `Appointments.jsx` (linked patient creation), `PatientViewSet.create` |
| **Read** | View patient list, individual patient history and all their appointments | `Appointments.jsx`, `PatientViewSet.list` / `.retrieve` |
| **Update** | Edit patient contact details and address | `Appointments.jsx`, `PatientViewSet.partial_update` |
| **Delete** | Remove a patient record from the system | `PatientViewSet.destroy` |

## Database Connectivity
- **Model:** `Appointment` — table `appointments` in `backend/api/models.py`  
  Fields: `patient_name`, `patient_age`, `patient_disease`, `contact_number`, `address`, `doctor` (FK), `appointment_date`, `appointment_time`, `booking_time`, `status`
- **Model:** `Patient` — table `patients` in `backend/api/models.py`  
  Fields: `name`, `age`, `phone_number`, `email`, `address`
- **Serializers:** `AppointmentSerializer`, `PatientSerializer` in `backend/api/serializers.py`
- **ViewSets:** `AppointmentViewSet`, `PatientViewSet` in `backend/api/views.py`
- **Hook:** `useAppointments.js` — React Query hook for fetching, creating, updating, deleting
- **Endpoints:**  
  - `GET  /api/appointments/` — List all appointments  
  - `POST /api/appointments/` — Create appointment  
  - `GET  /api/appointments/{id}/` — Get single appointment  
  - `PATCH /api/appointments/{id}/` — Update appointment  
  - `DELETE /api/appointments/{id}/` — Delete appointment  
  - `GET  /api/patients/` — List all patients  
  - `POST /api/patients/` — Create patient  
  - `PATCH /api/patients/{id}/` — Update patient  
  - `DELETE /api/patients/{id}/` — Delete patient
