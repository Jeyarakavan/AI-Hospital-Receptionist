## CRUD Operations Covered

### CRUD 1 — Ambulance / Emergency Request Management
| Operation | Description | Files Involved |
|-----------|-------------|----------------|
| **Create** | Submit an emergency ambulance request (caller name, location, condition) | `Ambulance.jsx`, `AmbulanceViewSet.create` in `views.py` |
| **Read** | View all ambulance requests — list with priority, status, timestamp | `Ambulance.jsx`, `AmbulanceViewSet.list` / `.retrieve` |
| **Update** | Update request status (Pending → Dispatched → Completed) | `Ambulance.jsx`, `AmbulanceViewSet.partial_update` |
| **Delete** | Remove resolved/cancelled ambulance request records | `Ambulance.jsx`, `AmbulanceViewSet.destroy` |

### CRUD 2 — Hospital News Management
| Operation | Description | Files Involved |
|-----------|-------------|----------------|
| **Create** | Admin publishes a hospital news broadcast/announcement | `News.jsx`, `hospital_news_create` view in `views.py` |
| **Read** | View published news list — title, body, author, publish date | `News.jsx`, `hospital_news_list` view in `views.py` |
| **Update** | Edit news content (via re-create workflow; news title/body update) | `News.jsx`, `send_hospital_news` view |
| **Delete** | Delete a news item by UUID | `News.jsx`, `hospital_news_delete` view in `views.py` |

## Database Connectivity
- **Model:** `Ambulance` — table `ambulance` in `backend/api/models.py`  
  Fields: caller name, location, condition description, priority, status, timestamp
- **Model:** Hospital News entries stored in the database via `hospital_news_create` / `hospital_news_list` / `hospital_news_delete` views in `views.py`
- **Model:** `ChatMessage` — table `chat_messages` — for internal messaging  
  Fields: sender (FK → User), recipient (FK → User), message body, timestamp, read status
- **ViewSet:** `AmbulanceViewSet`, `ChatMessageViewSet` in `backend/api/views.py`
- **Views:** `dashboard_stats`, `hospital_news_list`, `hospital_news_create`, `hospital_news_delete`, `send_hospital_news`, `send_message_to_user`, `chat_user_search` in `views.py`
- **Endpoints:**  
  - `GET  /api/ambulance/` — List ambulance requests  
  - `POST /api/ambulance/` — Create emergency request  
  - `PATCH /api/ambulance/{id}/` — Update status  
  - `DELETE /api/ambulance/{id}/` — Delete request  
  - `GET  /api/dashboard/` — Dashboard statistics  
  - `GET  /api/hospital-news/list/` — List news items  
  - `POST /api/hospital-news/create/` — Create news  
  - `DELETE /api/hospital-news/{id}/delete/` — Delete news  
  - `GET  /api/chat-messages/` — Chat messages  
  - `POST /api/send-message/` — Send chat message  
  - `GET  /api/chat-users/` — Search chat users
