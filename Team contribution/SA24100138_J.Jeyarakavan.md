## Function Overview
Responsible for the complete user identity layer of the system ‚Äî covering secure registration, login, profile management, and admin-level user administration. This function underpins every other module since all users (Admin, Doctor, Staff, Receptionist) must be authenticated before accessing the system.

## CRUD Operations Covered

### CRUD 1 ‚Äî User Registration & Account Management
| Operation | Description | Files Involved |
|-----------|-------------|----------------|
| **Create** | Register new users (Admin/Doctor/Staff/Receptionist) with role assignment | `Signup.jsx`, `AdminUserRegistration.jsx`, `AuthViewSet` in `views.py` |
| **Read** | View user list, individual profiles, pending approval requests | `UserManagement.jsx`, `Profile.jsx`, `UserViewSet` in `views.py` |
| **Update** | Edit profile details, change role/status, update profile picture | `Profile.jsx`, `AdminManage.jsx`, `UserViewSet.partial_update` |
| **Delete** | Deactivate or permanently remove user accounts | `UserManagement.jsx`, `UserViewSet.destroy` |

### CRUD 2 ‚Äî Admin User Approval Workflow
| Operation | Description | Files Involved |
|-----------|-------------|----------------|
| **Create** | Admin creates user accounts directly with pre-approved status | `AdminUserRegistration.jsx`, `AuthViewSet.register` |
| **Read** | List all pending/approved/rejected/disabled accounts | `AdminManage.jsx`, `UserViewSet.list` |
| **Update** | Approve, reject, or disable user accounts | `AdminManage.jsx`, `UserViewSet` ‚Äî status field update |
| **Delete** | Remove user records from the system | `UserManagement.jsx`, `UserViewSet.destroy` |

## Database Connectivity
- **Model:** `User` (custom `AbstractBaseUser`) ‚Äî table `users` in `backend/api/models.py`
- **Model:** `UserManager` ‚Äî custom manager for create_user / create_superuser
- **Serializer:** `UserSerializer`, `UserRegistrationSerializer` in `backend/api/serializers.py`
- **ViewSet:** `AuthViewSet` (login, register, logout, token refresh), `UserViewSet` (CRUD) in `backend/api/views.py`
- **Endpoints:**  
  - `POST /api/auth/register/` ‚Äî Create user  
  - `POST /api/auth/login/` ‚Äî JWT login  
  - `GET  /api/users/` ‚Äî List users  
  - `GET  /api/users/{id}/` ‚Äî Fetch single user  
  - `PATCH /api/users/{id}/` ‚Äî Update user  
  - `DELETE /api/users/{id}/` ‚Äî Delete user  
- **Auth:** JWT tokens (access + refresh) via SimpleJWT
