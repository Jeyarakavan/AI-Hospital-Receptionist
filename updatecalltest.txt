
AI Hospital Receptionist - Call Test Guide
Date: 2026-03-09

1) Start Backend (Windows)
- Open terminal at project root.
- Run:
  backend\venv\Scripts\python.exe backend/manage.py migrate
  backend\venv\Scripts\python.exe backend/manage.py runserver 0.0.0.0:8000

2) Backend Env Requirements
- In backend/.env, set at minimum:
  CALL_AI_SHARED_TOKEN=<same-token-used-by-agi>
- Keep call AI values from:
  backend/.env.call_ai.example

3) Deploy on Linux Asterisk Host
- Put project on Linux Asterisk server at:
  /opt/ai-hospital-receptionist
- Then run on Linux server (not Windows):
  chmod +x /opt/ai-hospital-receptionist/backend/call_ai/agi_entry.sh
- What this means:
  This script starts the AI receptionist when a call comes in.

4) Asterisk Dialplan Setup
- Copy file:
  backend/call_ai/asterisk/extensions_call_ai.conf
  -> /etc/asterisk/extensions_call_ai.conf
- In /etc/asterisk/extensions.conf add:
  #include "/etc/asterisk/extensions_call_ai.conf"

5) PJSIP Setup
- Merge snippets from:
  backend/call_ai/asterisk/pjsip_call_ai.conf
  into:
  /etc/asterisk/pjsip.conf
- Route inbound DID to context:
  ai-hospital-receptionist

6) Reload Asterisk
- Run:
  asterisk -rx "dialplan reload"
  asterisk -rx "pjsip reload"

7) Make a Real Test Call
- Call your inbound DID from mobile/landline.
- Expected:
  - AI answers and greets caller
  - Caller speaks
  - AI asks follow-up questions
  - AI asks confirmation before save
  - AI ends or escalates (handoff/emergency)

8) Verify Saved Records in Backend
- Login as authorized user and call:
  GET /api/call-ai/requests/
- Confirm fields are stored:
  call_id, caller_id, transcript, detected_intent,
  confidence_score, slots, confirmation_status,
  emergency_flag, handoff_flag, final_status

9) Manual Test Scenarios
- Booking: say booking intent + provide doctor/date/time + confirm yes
- Cancel: say cancel intent + provide details + confirm yes
- Reschedule: provide old/new details + confirm yes
- Emergency: say "chest pain" or "cannot breathe" (must escalate)
- Handoff: say "operator please" (must route handoff)
- Failed capture: unclear speech repeatedly (must mark failed_capture)

10) Useful Validation Commands
- Django checks:
  backend\venv\Scripts\python.exe backend/manage.py check
- Call AI API tests:
  backend\venv\Scripts\python.exe backend/manage.py test api.tests.test_call_ai_views
